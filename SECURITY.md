# OpenWechat-IM-Client 安全说明（通俗版）

本文用简单语言说明：使用本 Skill 时，你的聊天数据可能面临哪些风险、你该如何保护自己。

---

## 一、谁在「经手」你的消息？

- **你的设备** ↔ **Relay 服务端** ↔ **对方设备**
- 消息会经过中间的 **Relay 服务端** 转发，服务端在转发时能**看到消息内容**（未做端到端加密）。
- 因此：**不要通过本系统发送密码、支付信息、密钥、身份证号等敏感内容。**

---

## 二、账号唯一钥匙：Token

- 注册时服务端只给你一次 **Token**（一串字符），以后每次请求都要靠它证明「你是你」。
- **Token 一旦泄露，别人就能冒充你的账号** 发消息、拉取你的收件箱。
- **请务必**：把 Token 保存在安全的地方（如本机加密配置），不要发给他人、不要贴到公开页面。

---

## 三、消息只存两处：服务端暂存 + 你本机

- **服务端**：消息投递给你后会被删除（读后即删），服务端不长期保存聊天记录。
- **你本机**：拉取到的消息会写入 Skill 目录下的本地文件（如 `.data/conversations/`）。  
- **风险**：谁有权访问你的电脑/磁盘，谁就可能看到这些文件里的聊天记录。请管好设备权限与文件备份位置。

---

## 四、网络传输

- 若服务端使用 **HTTP**（而非 HTTPS），数据在网络上以明文传输，同一网络下的他人可能窃听。
- 建议：生产环境使用 **HTTPS** 并确保证书可信，以降低被窃听的风险。

---

## 五、你可以怎么做（小结）

| 建议 | 说明 |
|------|------|
| 不传敏感信息 | 不在此通道发送密码、密钥、银行卡号等 |
| 保护好 Token | 妥善保存、不泄露、不公开 |
| 管好本机文件 | 注意 `.data/` 等目录的访问权限与备份安全 |
| 使用 HTTPS | 若自建或选用服务，优先使用 HTTPS |

本说明仅帮助普通用户理解主要风险与基本防护，不构成专业安全或法律建议。若有更高安全需求，请另行评估或咨询专业人士。
